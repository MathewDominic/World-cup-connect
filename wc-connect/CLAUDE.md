# wc-connect

Standalone browser game: connect two footballers through shared FIFA World Cup squad appearances.

## What this does

Players are given a **start player** and a **goal player** from the same country's World Cup history. To connect them, the player clicks through a chain of shared WC squads:

1. The start player is shown → pick which World Cup year they appeared in
2. The full squad for that year is shown → pick a teammate
3. Repeat until you reach the goal player

The intro screen shows "Can you connect?" with both players' photos and the country flag, matching the style of the Connect Remotion video.

## Running

No build step. Open `index.html` directly in a browser.

For a local HTTP server (avoids any file:// quirks):
```bash
npx serve .
# or
python3 -m http.server 8080
```

## Generating squad data

```bash
node scripts/scrape-squads.mjs
```

Two phases:
1. **Squad scraping** — fetches Wikipedia squad template pages for ~48 nations × years 1990–2026 (~3–5 min)
2. **Pageviews** — fetches 12 months of Wikipedia pageviews for every unique player (~5–8 min, ~20 req/s)

Outputs:
- `data/squads.js` — JS global with both `window.WC_SQUADS` and `window.WC_PLAYER_POPULARITY`
- `data/squads.json` — squads only
- `data/popularity.json` — player name → annual pageview count

Result: ~240 squads, ~3000 unique players with popularity scores

## Key facts

- **No build step, no backend, no database** — plain HTML + vanilla JS
- `data/squads.js` is loaded as a `<script>` tag; sets `window.WC_SQUADS`
- All game logic runs in-browser from the JSON data
- Player photos fetched lazily from Wikipedia REST API (`page/summary` endpoint)
- Photo URL is used as-is from the thumbnail — no size modification (avoids 400 errors when original < requested size)
- **Same-country chains only** — BFS stays within one nation's WC history

## Game data model

```
squads.js → window.WC_SQUADS = [
  { country: "England", year: 2022, players: ["Harry Kane", "Jude Bellingham", ...] },
  ...
]
```

In-browser bipartite graph built at load time:
- `playerToSquads`: "Harry Kane" → [{country, year, key}, ...]
- `squadToPlayers`: "England_2022" → ["Harry Kane", ...]
- BFS alternates player ↔ squad nodes, filtered by country

## Player popularity

`window.WC_PLAYER_POPULARITY` is a flat `{ "Player Name": totalAnnualViews }` map generated by the scraper. Used for:

1. **Puzzle generation** — both endpoints are drawn from the top 30 most-viewed players for that country (falling back to top 50% for shorter paths). This ensures puzzles feature recognisable players.
2. **Squad display** — players in the pick-player grid are sorted by popularity descending, so famous players appear first and are easy to spot.

Falls back to `0` if the player has no Wikipedia pageviews data.

## Analytics

PostHog, loaded via snippet in `index.html`. Set `window.POSTHOG_API_KEY` (and `window.POSTHOG_HOST` for EU cloud) at the top of the snippet — analytics no-ops entirely when the key is empty. Custom events are sent through the `track()` helper in `game.js`: `game_started`, `game_won`, `game_retried`, `path_revealed`, `share_clicked`, `share_link_opened`, `country_selected`. Pageviews/autocapture come from the snippet defaults.

## Country selection

Countries are picked randomly with weights favouring top footballing nations:
- Brazil / Argentina: 20×
- Germany / France / Spain: 18×
- England: 16×, Portugal: 14×, Netherlands: 12×
- Smaller nations: 1–8×

Only countries with ≥20 known players are eligible.

## Wikipedia scraper — parsing strategies

The scraper tries three strategies in order per page, falling back if a strategy finds fewer than 11 players:

1. `|name=[[Player Name]]` — standard `{{Nat fs player}}` template
2. `|p1 = [[Player Name|display]]` — `#invoke:national squad` style (Brazil, Portugal)
3. General `|name=` / `|player=` fields
4. Last resort: `[[Firstname Lastname]]` wikilinks that look like people names

Name cleaning: strips disambiguation suffixes like `(footballer, born 1963)`.

## Wikitext parsing edge cases

- `cleanName()` strips ` (disambiguation)` suffixes from player names
- `isValidPlayer()` filters out category links, file links, role names (captain, coach, etc.)
- `SKIP_NAMES` regex catches common non-player entries
- Countries with aliases (e.g. `Germany` / `West Germany` for 1990) try each alias in order
